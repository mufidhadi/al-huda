# Development Log: Tanya Quran Hadist

**Project Name:** Tanya Quran Hadist  
**Date:** 25 Maret 2026  
**Developer:** Gemini CLI & Mas Mufid  

## 🚀 Project Overview
Membangun sistem kueri dan dataset terpadu untuk teks suci Al-Quran dan Hadits. Fokus utama adalah pada integritas data, pemisahan struktural (Sanad vs Matan), dan optimasi performa penyerapan data menggunakan infrastruktur modern. Kini telah mendukung kueri semantik lintas sumber dengan library mesin pencari terpadu.

## 🛠️ Tech Stack
- **Python 3.12**: Bahasa pemrograman inti.
- **UV**: Manajer paket dan proyek tercepat.
- **PostgreSQL (v16) + pgvector**: Basis data relasional dengan kapabilitas kueri semantik.
- **Sentence-Transformers (E5-Small)**: Model lokal untuk pemrosesan teks menjadi vektor 384 dimensi.
- **pytest**: Framework pengujian otomatis.
- **tqdm**: Visualisasi progres proses intensif.

## 📝 Experience & Task List

| Task | Issue | Tried Approach | Result | Lesson Learned (Project) | Lesson Learned (General Dev) | Lesson Learned (AI Prompting) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Quran Ingestion** | Duplikasi data karena proses paralel tumpang tindih. | TRUNCATE sederhana. | Data terduplikasi. | Domain data religi sangat sensitif terhadap jumlah baris. | Selalu lakukan *cleanup* proses background sebelum operasi ETL. | AI membutuhkan instruksi pembersihan proses eksplisit. |
| **Gap Analysis & Resume** | Koneksi terputus di tengah proses besar. | Pengecekan status DB per baris. | Ingestion cerdas. | Dataset Al-Quran butuh validasi jumlah ayat per surah. | Implementasikan *Idempotency* pada setiap fungsi penulisan data. | Berikan AI konteks arsitektur "Smart Resume". |
| **Sanad-Matan Split** | Struktur teks hadits tercampur aduk. | Algoritma *Rightmost Search* (V3). | Akurasi pemisahan >95%. | Sanad dan Matan memiliki pola transisi linguistik yang dinamis. | Pendekatan *Positional Split* lebih tangguh daripada *Keyword Match* statis. | Gunakan sampling audit untuk menguji limitasi logika AI secara forensik. |
| **Vector Embedding** | Butuh pencarian makna (Semantic Search). | Local E5-Small model + pgvector. | 100% Independent AI infra. | E5-Small adalah model optimal untuk pencarian asimetris. | Shared Primary Key pada tabel embedding meningkatkan performa JOIN. | Jelaskan konsep "Independence" model (cached) agar pengguna merasa aman. |
| **Unified Search Engine** | Perlunya logika pencarian yang konsisten untuk berbagai interface. | Membangun library `QuranHadithSearch` dengan perankingan berbasis Cosine Distance. | Pencarian lintas sumber (Quran & Hadits) berhasil dengan format output seragam. | Abstraksi logika pencarian ke dalam library memudahkan skalabilitas UI di masa depan. | Gunakan pola Singleton/Shared instance untuk model AI agar hemat RAM. | Deskripsikan alur pencarian "Google-style" untuk mengarahkan AI membuat perankingan relevansi. |
| **Massive Testing** | Memastikan reliabilitas pada ratusan kemungkinan kueri. | Implementasi 201 test case menggunakan `pytest.mark.parametrize`. | 200/201 Lolos (1 teknis timeout). | Kueri semantik sangat tangguh menangani variasi bahasa manusia. | Pengujian skala besar pada database remote rentan terhadap *network jitter/timeout*. | Minta AI membuat daftar kueri yang variatif (teologis, hukum, adab) untuk cakupan tes yang luas. |
