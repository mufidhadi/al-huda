# Development Log: Tanya Quran Hadist

**Project Name:** Tanya Quran Hadist  
**Date:** 25 Maret 2026  
**Developer:** Gemini CLI & Mas Mufid  

## 🚀 Project Overview
Membangun sistem kueri dan dataset terpadu untuk teks suci Al-Quran dan Hadits. Fokus utama adalah pada integritas data, pemisahan struktural (Sanad vs Matan), dan optimasi performa penyerapan data menggunakan infrastruktur modern. Kini telah mendukung kueri semantik lintas sumber dan memiliki blueprint desain UI/UX yang meniru pengalaman Google Search original.

## 🛠️ Tech Stack
- **Python 3.12**: Bahasa pemrograman inti.
- **UV**: Manajer paket dan proyek tercepat.
- **PostgreSQL (v16) + pgvector**: Basis data relasional dengan kapabilitas kueri semantik.
- **Sentence-Transformers (E5-Small)**: Model lokal untuk pemrosesan teks menjadi vektor 384 dimensi.
- **Stitch**: Alat desain untuk prototipe UI/UX mobile-friendly.
- **pytest & psutil**: Framework pengujian dan monitoring performa.

## 📝 Experience & Task List

| Task | Issue | Tried Approach | Result | Lesson Learned (Project) | Lesson Learned (General Dev) | Lesson Learned (AI Prompting) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Parsing Logic V3** | Kasus khusus (ID 4, 5, dsb) gagal dipisah. | Implementasi *Rightmost Search* (`rfind`) untuk mencari perawi terakhir. | Akurasi pemisahan >95% untuk Bukhari. | Struktur hadits sangat dinamis; posisi tanda baca adalah kunci pemisah terbaik. | Pendekatan *Positional Split* lebih tangguh daripada *Keyword Match* statis. | Gunakan sampling audit forensik untuk menguji logika. |
| **Vector Embedding** | Butuh pencarian makna (Semantic Search). | Local E5-Small model + pgvector. | 100% Independent AI infra. | E5-Small optimal untuk pencarian asimetris Quran/Hadits. | Shared Primary Key pada tabel embedding meningkatkan performa JOIN. | Jelaskan konsep "Independent Model" (cached) kepada pengguna. |
| **Unified Search** | Perlunya logika pencarian konsisten untuk UI. | Library `QuranHadithSearch` dengan Cosine Distance ranking. | Pencarian lintas sumber berhasil dengan format output seragam. | Abstraksi logika memudahkan skalabilitas UI di masa depan. | Gunakan pola Singleton untuk model AI agar hemat RAM. | Deskripsikan alur "Google-style" untuk perankingan. |
| **Performance Audit** | Belum diketahui kebutuhan resource riil. | Benchmarking latensi & RAM usage via `psutil`. | Latensi <1s, RAM ~325MB. | E5-Small sangat ramah untuk VPS resource rendah. | Latensi didominasi network I/O pada database remote. | Instruksikan pembuatan laporan formal untuk dokumentasi. |
| **UI/UX Design** | Butuh antarmuka yang familiar dan bersih. | Replikasi Google Search Original menggunakan Stitch. | Desain "Pure Search" yang minimalis dan mobile-friendly. | Antarmuka yang familiar (Google-style) meminimalkan kurva pembelajaran user. | Filosofi "Invisibility of UI" mengutamakan utilitas di atas dekorasi. | Gunakan instruksi "Strict Replica" untuk mengarahkan AI membuat desain yang konsisten. |
| **Social Share** | Perlunya fitur awareness tanpa mengganggu UX. | Server-side image rendering (1:1) dan text copy dengan opsi branding toggle. | Pengguna dapat share konten estetik dengan kontrol penuh privasi. | Memberikan opsi "Hide Branding" membangun kepercayaan pengguna terhadap aplikasi. | Pisahkan fitur interaktif (Share) dari alur pencarian cepat (Result Page). | Definisikan skema sharing dalam API Contract sebelum mendesain UI. |
