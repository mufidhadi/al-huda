# Development Log: Tanya Quran Hadist

**Project Name:** Tanya Quran Hadist  
**Date:** 25 Maret 2026  
**Developer:** Gemini CLI & Mas Mufid  

## 🚀 Project Overview
Membangun sistem kueri dan dataset terpadu untuk teks suci Al-Quran dan Hadits. Fokus utama adalah pada integritas data, pemisahan struktural (Sanad vs Matan), dan optimasi performa penyerapan data menggunakan infrastruktur modern. Kini telah mendukung kueri semantik lintas sumber dengan library mesin pencari terpadu yang telah diaudit performanya.

## 🛠️ Tech Stack
- **Python 3.12**: Bahasa pemrograman inti.
- **UV**: Manajer paket dan proyek tercepat.
- **PostgreSQL (v16) + pgvector**: Basis data relasional dengan kapabilitas kueri semantik.
- **Sentence-Transformers (E5-Small)**: Model lokal untuk pemrosesan teks menjadi vektor 384 dimensi.
- **pytest**: Framework pengujian otomatis.
- **psutil**: Monitoring resource sistem.
- **tqdm**: Visualisasi progres proses intensif.

## 📝 Experience & Task List

| Task | Issue | Tried Approach | Result | Lesson Learned (Project) | Lesson Learned (General Dev) | Lesson Learned (AI Prompting) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Quran Ingestion** | Duplikasi data karena proses paralel tumpang tindih. | TRUNCATE sederhana. | Data terduplikasi. | Domain religi sensitif terhadap jumlah baris. | Selalu lakukan *cleanup* proses background sebelum operasi ETL. | AI butuh instruksi pembersihan eksplisit. |
| **Gap Analysis & Resume** | Koneksi terputus di tengah proses besar. | Pengecekan status DB per baris. | Ingestion cerdas. | Dataset Quran butuh validasi jumlah ayat per surah. | Implementasikan *Idempotency* pada setiap fungsi penulisan data. | Berikan AI konteks arsitektur "Smart Resume". |
| **Sanad-Matan Split** | Struktur teks hadits tercampur aduk. | Algoritma *Rightmost Search* (V3). | Akurasi pemisahan >95%. | Sanad dan Matan memiliki pola dinamis. | Pendekatan *Positional Split* lebih tangguh daripada *Keyword Match* statis. | Gunakan sampling audit forensik untuk menguji logika. |
| **Vector Embedding** | Butuh pencarian makna (Semantic Search). | Local E5-Small model + pgvector. | 100% Independent AI infra. | E5-Small optimal untuk pencarian asimetris. | Shared Primary Key pada tabel embedding meningkatkan performa JOIN. | Jelaskan konsep "Independent Model" (cached) kepada pengguna. |
| **Unified Search Engine** | Perlunya logika pencarian konsisten. | Library `QuranHadithSearch` dengan Cosine Distance. | Pencarian lintas sumber berhasil. | Abstraksi logika memudahkan skalabilitas UI di masa depan. | Gunakan pola Singleton untuk model AI agar hemat RAM. | Deskripsikan alur "Google-style" untuk perankingan. |
| **Performance Audit** | Belum diketahui kebutuhan resource riil. | Benchmarking latensi & RAM usage via `psutil`. | Latensi <1s, RAM ~325MB. | E5-Small sangat ramah untuk VPS resource rendah (RAM 1-2GB). | Latensi didominasi network I/O pada database remote. | Instruksikan pembuatan laporan formal untuk dokumentasi keputusan arsitektur. |

## 🏁 Summary
Proyek **Tanya Quran Hadist** telah mencapai tahap **Production-Ready Core**. Kita telah memiliki 12.874 data terindeks vektor, library mesin pencari yang divalidasi oleh 200+ test case, dan profil performa yang sangat efisien (latency <1s). Infrastruktur ini 100% independen dan siap menjadi fondasi berbagai interface aplikasi masa depan.
