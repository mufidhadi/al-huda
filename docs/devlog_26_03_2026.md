# 🚀 DevLog: Security Hardening & Documentation Standard
**Date:** 26 March 2026
**Project:** al-Huda (Tanya Quran Hadist)
**Developer:** Gemini CLI & Mas Mufid

## 📝 Narrative Summary
Hari ini fokus utama adalah pada **Security Hardening** (Pengerasan Keamanan) dan **Standardisasi Proyek**. Kita menyadari bahwa kredensial database yang sebelumnya tertulis langsung (*hardcoded*) di dalam kode sumber merupakan risiko keamanan yang besar, terutama jika proyek ini nantinya di-commit ke repositori publik. 

Proses dimulai dengan memigrasi seluruh kredensial ke file `.env` dan memastikan riwayat Git dibersihkan dari jejak-jejak rahasia tersebut menggunakan `git-filter-repo`. Selain itu, kita juga melengkapi dokumentasi teknis dengan file `.env.example` dan `README.md` yang lebih informatif untuk memudahkan kolaborasi di masa depan.

## 🛠️ Tech Stack Updates
- **python-dotenv**: Ditambahkan untuk manajemen variabel lingkungan.
- **git-filter-repo**: Digunakan sebagai instrumen pembersih riwayat Git yang presisi.

## 📊 Task & Experiences
| Task | Issue | Tried Approach | Result | Lesson Learned (Project) | General SE Lesson | AI Prompting Lesson |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **Credential Migration** | Kredensial database masih hardcoded di 11+ file Python. | Migrasi ke `.env` dan gunakan `os.getenv()`. | Seluruh file kini bersih dari rahasia langsung. | Sentralisasi konfigurasi memudahkan penggantian infrastruktur (Remote vs Local DB). | Jangan pernah membiarkan *secrets* berada di kode sumber, meski hanya untuk testing. | Gunakan `generalist` agent untuk melakukan *bulk refactor* yang membosankan tapi krusial. |
| **Git History Cleaning** | Jejak kredensial lama masih tersimpan di riwayat commit Git. | Gunakan `git-filter-repo` dengan pemetaan string placeholder. | Riwayat Git bersih; kredensial lama digantikan oleh placeholder di seluruh sejarah commit. | Membersihkan rahasia di Git butuh ketelitian agar file konfigurasi yang sedang aktif tidak ikut terhapus. | `git filter-branch` sudah dianggap usang; `git-filter-repo` jauh lebih cepat dan aman. | Pastikan file bantuan (*replacements.txt*) tidak ikut ter-track di Git sebelum proses pembersihan. |
| **Environment Sync** | Ada variabel Qdrant di `.env` padahal proyek murni menggunakan pgvector. | Sinkronisasi manual `.env` dan `.env.example` berdasarkan `docker-compose.yml`. | Konfigurasi `.env` kini 100% akurat sesuai kebutuhan backend & frontend. | Selalu verifikasi ulang dependensi di `pyproject.toml` untuk menghindari variabel lingkungan yang tidak perlu. | Dokumentasi yang sinkron dengan kode adalah bentuk komunikasi terbaik antar developer. | Berikan koreksi langsung jika AI berasumsi terlalu jauh (misal: berasumsi pakai Qdrant). |
| **Traefik Integration** | Butuh akses via domain publik al-huda.masmuf.cloud. | Menambahkan labels Traefik dan network eksternal `web` di `docker-compose.yml`. | Aplikasi kini siap diakses via HTTPS di domain al-huda.masmuf.cloud. | Penggunaan network eksternal `web` sangat krusial untuk komunikasi antar kontainer di bawah satu reverse proxy (Traefik). | Gunakan label `traefik.http.services.<name>.loadbalancer.server.port` jika port kontainer bukan port default (80). | Mintalah contoh dari proyek lain yang sudah stabil untuk mempercepat konfigurasi infrastruktur. |
| **CORS Configuration** | Frontend diblokir oleh browser saat memanggil API lintas domain. | Implementasi `CORSMiddleware` di FastAPI dan update URL produksi di frontend. | Komunikasi antar domain `al-huda.masmuf.cloud` dan `api-alhuda.masmuf.cloud` berjalan lancar. | Dalam server produksi, CORS harus dikonfigurasi secara eksplisit untuk domain yang diizinkan guna keamanan. | Selalu gunakan HTTPS di environment produksi untuk menghindari isu *mixed content*. | Periksa log browser (Network tab) untuk mendeteksi kegagalan request akibat CORS atau DNS. |
| **Backend Stability Fix** | Kontainer backend mengalami *crash loop* (502 Bad Gateway) karena server tidak dipanggil secara eksplisit. | Mengubah `CMD` di `Dockerfile.backend` untuk menjalankan `uvicorn` secara langsung dengan host `0.0.0.0`. | Server backend kini stabil, tidak *exit* setelah memuat model AI, dan dapat diakses oleh Traefik. | Pastikan `CMD` Docker menjalankan proses yang *long-running* agar kontainer tidak dianggap selesai dan melakukan restart otomatis. | Gunakan `--host 0.0.0.0` agar server di dalam kontainer dapat menerima trafik dari luar (Traefik). | Jangan hanya mengandalkan `docker ps`, periksa durasi `Up` untuk mendeteksi *crash loop*. |
| **Frontend URL Fix** | Halaman detail (Hadits/Quran) masih memanggil `localhost:8000`. | Mengganti seluruh URL hardcoded dengan `NEXT_PUBLIC_API_URL` dan memperbaiki kesalahan penamaan variabel (`json` vs `result`). | Seluruh fitur frontend kini berkomunikasi dengan API produksi secara konsisten. | Selalu gunakan pencarian global (grep) untuk memastikan tidak ada URL hardcoded yang tertinggal saat migrasi ke produksi. | Sinkronisasi nama variabel antara blok `try-catch` sangat krusial dalam JavaScript/TypeScript untuk menghindari build error. | Berhati-hatilah saat melakukan *find & replace* massal agar tidak merusak struktur logika kode. |
| **Image Generator Font Fix** | Karakter Arab ter-render sebagai "tofu" (kotak kosong) karena masalah font di Docker. | Bundling font Noto Sans & Noto Sans Arabic langsung ke dalam repositori dan menyalinnya ke kontainer via `Dockerfile`. | Gambar share kini merender teks Arab dan Indonesia dengan sempurna dan terbaca jelas. | Mengandalkan `apt-get` untuk install font di Docker bisa gagal jika mirror Debian tidak stabil; bundling font ke project jauh lebih andal (*deterministic*). | Font Noto Sans adalah pilihan terbaik untuk dukungan karakter multibahasa yang luas dan konsisten. | Selalu tes hasil render gambar di environment produksi, karena ketersediaan font sistem sangat berbeda antar OS. |
| **Pagination Implementation** | Hasil pencarian hanya menampilkan 10 item pertama tanpa navigasi halaman. | Menambahkan parameter `page` pada request API dan membangun komponen UI navigasi halaman di frontend. | Pengguna kini dapat menelusuri hasil pencarian hingga 10 halaman (100 hasil) dengan navigasi yang intuitif. | Metadata dari backend (total_results, per_page) sangat membantu untuk menghitung jumlah tombol halaman secara dinamis di frontend. | Gunakan `window.scrollTo` agar saat pindah halaman, tampilan otomatis kembali ke atas untuk UX yang lebih baik. | Pastikan parameter query di URL tersinkronisasi dengan *state* aplikasi agar fitur "Back" browser tetap bekerja sesuai harapan. |
| **VPS Deployment** | Project harus segera tersedia di server produksi. | Menggunakan MCP SSH untuk clone repo dan manual setup `.env` di VPS. | Project al-Huda kini sudah ter-deploy di direktori `project/al-huda/` di VPS. | Selalu cek konsistensi struktur direktori antar project di VPS agar manajemen *deployment* lebih teratur. | Gunakan perintah `printf` atau metode *safe write* lainnya saat mengedit file via SSH-MCP untuk menghindari isu `here-document`. | SSH-MCP sangat membantu untuk otomasi deployment tanpa harus keluar dari lingkungan kerja utama. |

## 🏁 Final Thought
Dengan infrastruktur yang kini aman dan terdokumentasi dengan baik, **al-Huda** telah memenuhi standar profesional untuk pengembangan perangkat lunak modern. Langkah selanjutnya adalah memastikan sinkronisasi antara backend ini dengan implementasi frontend yang akan dibangun.
